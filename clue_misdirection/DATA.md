# Data Dictionary ‚Äî Clue Misdirection

## Source Data

### George Ho's CCC Dataset

- **Location:** `../data/data.sqlite3` (shared with indicator_clustering)
- **Download:** https://cryptics.georgeho.org/data.db (saves as `data.sqlite3`)
- **License:** Open Database License
- **Size:** 660,613 cryptic crossword clues
- **Key columns:**
  - `clue` ‚Äî Full clue text including answer format, e.g., "Seed scattered in a
    simple device (6)"
  - `answer` ‚Äî The answer word, e.g., "DIODES"
  - `definition` ‚Äî The definition substring within the clue, e.g., "simple device".
    For double-definition clues, contains `/`-separated alternatives.
  - `wordplay_type` ‚Äî Type label (anagram, reversal, etc.) ‚Äî not always present
  - `indicator` ‚Äî Indicator word/phrase ‚Äî not always present
  - `source_url`, `source`, `date` ‚Äî Provenance metadata

### Raw CSV Extract

- **Location:** `../data/clues_raw.csv`
- **Description:** Extracted from the sqlite DB by the indicator_clustering
  NB00. Contains the same columns as the sqlite `clues` table. This is our
  starting point ‚Äî we do not go back to the sqlite directly.

## Intermediate Data (Generated by Pipeline)

All intermediate files are written to `clue_misdirection/data/`.

### Step 1 Output

| File | Description |
|------|-------------|
| `data/clues_filtered.csv` | Rows passing all Section 3.2 filters |
| `data/cleaning_log.md` | Record of row counts at each filter step |

**Schema for `clues_filtered.csv`:**

| Column | Type | Description |
|--------|------|-------------|
| `clue_id` | int | Unique identifier (from source) |
| `clue` | str | Full clue text including answer format, e.g., "Plant in a garden party (5)" |
| `surface` | str | Clue text with answer format stripped, e.g., "Plant in a garden party". **This is what we embed** ‚Äî never the raw `clue`. |
| `surface_normalized` | str | Lowercase, no punctuation/accents version of `surface` (used for validation) |
| `definition` | str | The definition substring (single definition for this row) |
| `answer` | str | The answer word or phrase (uppercase) |
| `answer_format` | str | Format string extracted from clue, e.g., "7" or "3,4" |
| `num_definitions` | int | Number of valid definitions found for this clue (1 for most rows; >1 if the clue had multiple valid definitions and was expanded into multiple rows) |
| `def_answer_pair_id` | int | Unique ID for each (definition, answer) pair ‚Äî used for GroupKFold |

**Actual counts:** 241,397 rows, 129,429 unique (definition, answer) pairs.
`wordplay_type` was not available in `clues_raw.csv` and is excluded from the
output (consistent with Decision 4). Article stripping ("a ") was applied
during WordNet lookup to recover additional matches; further lemmatization is
a potential future improvement.

### Step 2 Output

Embeddings are deduplicated: definition and answer embeddings are computed
once per unique string; only clue-context embeddings are computed per row.
See PLAN.md Step 2 for the full rationale.

| File | Shape | Description |
|------|-------|-------------|
| `data/embeddings/definition_embeddings.npy` | (N_def, 3, D) | Per unique definition: [average, common, obscure] |
| `data/embeddings/definition_index.csv` | (N_def, 2) | Maps row position ‚Üí definition string |
| `data/embeddings/answer_embeddings.npy` | (N_ans, 3, D) | Per unique answer: [average, common, obscure] |
| `data/embeddings/answer_index.csv` | (N_ans, 2) | Maps row position ‚Üí answer string |
| `data/embeddings/clue_context_embeddings.npy` | (N_rows, 2, D) | Per clue row: [word1_clue_context, sentence1] |
| `data/embeddings/clue_context_index.csv` | (N_rows, 2) | Maps row position ‚Üí clue_id |

- **N_def** = number of unique definition strings in `clues_filtered.csv`
- **N_ans** = number of unique answer strings in `clues_filtered.csv`
- **N_rows** = number of rows in `clues_filtered.csv`
- **D** = embedding dimension (768 for CALE/ModernBERT-base)
- The three slots in definition/answer embeddings correspond to:
  `[0]` = average (no context), `[1]` = common synset, `[2]` = obscure synset

**Mapping embeddings to clue rows:** For a given row in `clues_filtered.csv`,
look up its definition string in `definition_index.csv` to get the row
position in `definition_embeddings.npy`, and similarly for the answer. The
clue-context embeddings are aligned directly with `clues_filtered.csv` row
order.

### Step 3 Output

| File | Description |
|------|-------------|
| `data/features_all.parquet` | All 53 features + metadata for every clue row |

**Feature columns (53 total):**

**Context-Free Meaning (15):** Cosine similarities among embeddings that don't
involve clue context. Named as `cos_{emb_a}_{emb_b}`, e.g.:
- `cos_w1avg_w2avg`, `cos_w1avg_w2common`, `cos_w1avg_w2obscure`
- `cos_w1common_w2avg`, `cos_w1common_w2common`, `cos_w1common_w2obscure`
- `cos_w1obscure_w2avg`, `cos_w1obscure_w2common`, `cos_w1obscure_w2obscure`
- `cos_w1avg_w1common`, `cos_w1avg_w1obscure`, `cos_w1common_w1obscure`
- `cos_w2avg_w2common`, `cos_w2avg_w2obscure`, `cos_w2common_w2obscure`

**Context-Informed Meaning (13):** Any cosine similarity involving
`word1_clue_context` or `sentence1`:
- `cos_w1clue_w1avg`, `cos_w1clue_w1common`, `cos_w1clue_w1obscure`
- `cos_w1clue_w2avg`, `cos_w1clue_w2common`, `cos_w1clue_w2obscure`
- `cos_w1clue_sent1`
- `cos_sent1_w1avg`, `cos_sent1_w1common`, `cos_sent1_w1obscure`
- `cos_sent1_w2avg`, `cos_sent1_w2common`, `cos_sent1_w2obscure`

**Relationship (21):**
- 19 boolean columns for two-hop WordNet relationship types (e.g.,
  `rel_synonym`, `rel_hyponym`, `rel_hypernym`, `rel_hyponym_of_hypernym`, etc.)
- `rel_max_path_similarity` (float, default 0.0)
- `rel_shared_synset_count` (int, default 0)

**Surface (4):**
- `surf_edit_distance` (int)
- `surf_length_ratio` (float)
- `surf_shared_first_letter` (bool/int)
- `surf_char_overlap_ratio` (float)

**Metadata columns (not features):**
- `clue_id`, `clue`, `surface`, `definition`, `answer`, `def_answer_pair_id`
