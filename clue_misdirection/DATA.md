# Data Dictionary — Clue Misdirection

## Source Data

### George Ho's CCC Dataset

- **Location:** `../data/data.sqlite3` (shared with indicator_clustering)
- **Download:** https://cryptics.georgeho.org/data.db (saves as `data.sqlite3`)
- **License:** Open Database License
- **Size:** 660,613 cryptic crossword clues
- **Key columns:**
  - `clue` — Full clue text including answer format, e.g., "Seed scattered in a
    simple device (6)"
  - `answer` — The answer word, e.g., "DIODES"
  - `definition` — The definition substring within the clue, e.g., "simple device".
    For double-definition clues, contains `/`-separated alternatives.
  - `wordplay_type` — Type label (anagram, reversal, etc.) — not always present
  - `indicator` — Indicator word/phrase — not always present
  - `source_url`, `source`, `date` — Provenance metadata

### Raw CSV Extract

- **Location:** `../data/clues_raw.csv`
- **Description:** Extracted from the sqlite DB by the indicator_clustering
  NB00. Contains the same columns as the sqlite `clues` table. This is our
  starting point — we do not go back to the sqlite directly.

## Intermediate Data (Generated by Pipeline)

All intermediate files are written to `clue_misdirection/data/`.

### Step 1 Output

| File | Description |
|------|-------------|
| `data/clues_filtered.csv` | Rows passing all Section 3.2 filters |
| `data/cleaning_log.md` | Record of row counts at each filter step |

**Schema for `clues_filtered.csv`:**

| Column | Type | Description |
|--------|------|-------------|
| `clue_id` | int | Unique identifier (from source) |
| `clue` | str | Full clue text including answer format, e.g., "Plant in a garden party (5)" |
| `surface` | str | Clue text with answer format stripped, e.g., "Plant in a garden party". **This is what we embed** — never the raw `clue`. |
| `surface_normalized` | str | Lowercase, no punctuation/accents version of `surface` (used for validation) |
| `definition` | str | The definition substring (single definition for this row) |
| `answer` | str | The answer word or phrase (uppercase) |
| `answer_format` | str | Format string extracted from clue, e.g., "7" or "3,4" |
| `num_definitions` | int | Number of valid definitions found for this clue (1 for most rows; >1 if the clue had multiple valid definitions and was expanded into multiple rows) |
| `def_answer_pair_id` | int | Unique ID for each (definition, answer) pair — used for GroupKFold |

**Actual counts:** 241,397 rows, 129,429 unique (definition, answer) pairs.
`wordplay_type` was not available in `clues_raw.csv` and is excluded from the
output (consistent with Decision 4). Article stripping ("a ") was applied
during WordNet lookup to recover additional matches; further lemmatization is
a potential future improvement.

### Step 2 Intermediate Files (Phrase Construction)

These CSV files are generated by the CPU portion of `02_embedding_generation.ipynb`
(cells 0–21) and consumed by the GPU portion. They are not committed to the repo
(listed in `.gitignore`) but are documented here for reference.

| File | Rows | Description |
|------|------|-------------|
| `data/embeddings/definition_phrases.csv` | ~100K | One row per (unique_definition, synset) pair. Each row contains a CALE-ready phrase with `<t></t>` delimiters. |
| `data/embeddings/answer_phrases.csv` | ~127K | Same structure for unique answers. |
| `data/embeddings/clue_context_phrases.csv` | 240,211 | One row per clue. Contains the surface text with `<t></t>` around the definition, plus metadata for downstream joins. |

**Schema for `definition_phrases.csv` and `answer_phrases.csv`:**

| Column | Type | Description |
|--------|------|-------------|
| `word` | str | The definition or answer string used for WordNet lookup (lowercased, underscored, article-stripped where needed) |
| `synset_name` | str | WordNet synset identifier (e.g., "plant.n.01") |
| `synset_idx` | int | 0-based index of this synset among all synsets for the word |
| `num_synsets` | int | Total number of WordNet synsets for this word (before filtering) |
| `is_common` | bool | True if this is the most frequent sense (synset_idx == 0) |
| `is_obscure` | bool | True if this is the least frequent sense (synset_idx == num_synsets - 1). For single-synset words, both is_common and is_obscure are True. |
| `phrase` | str | The CALE-ready context sentence with `<t></t>` around the target word |
| `path` | str | Which construction method was used: "example", "definition", or "fallback_a" |
| `num_usable_synsets` | int | Number of synsets that produced valid phrases for this word (after removing unresolvable ones). When this equals 1, allsense = common = obscure for this word. |

**Schema for `clue_context_phrases.csv`:**

| Column | Type | Description |
|--------|------|-------------|
| `clue_id` | int | Unique clue identifier from clues_filtered.csv |
| `definition` | str | Original definition string (for surface matching) |
| `definition_wn` | str | WordNet-ready definition (lowercased, underscored, article-stripped) |
| `answer` | str | Original answer string (ALL-CAPS) |
| `answer_wn` | str | WordNet-ready answer (lowercased, underscored, article-stripped) |
| `surface` | str | Clue text without answer format |
| `clue_context_phrase` | str | Surface text with `<t></t>` around the definition |
| `def_num_usable_synsets` | int | Number of usable synsets for this row's definition_wn |
| `ans_num_usable_synsets` | int | Number of usable synsets for this row's answer_wn |

### Step 2 Output

Embeddings are deduplicated: definition and answer embeddings are computed
once per unique string; only clue-context embeddings are computed per row.
See PLAN.md Step 2 for the full rationale.

All embeddings are generated using `gabrielloiseau/CALE-MBERT-en` with
`<t></t>` delimiters around the target word in a context sentence.

| File | Shape | Size | Description |
|------|-------|------|-------------|
| `data/embeddings/definition_embeddings.npy` | (27,385, 3, 1024) | 321 MB | Per unique definition: [allsense_avg, common, obscure] |
| `data/embeddings/definition_index.csv` | (27,385, 2) | <1 MB | Maps row position → definition string |
| `data/embeddings/answer_embeddings.npy` | (45,254, 3, 1024) | 530 MB | Per unique answer: [allsense_avg, common, obscure] |
| `data/embeddings/answer_index.csv` | (45,254, 2) | <1 MB | Maps row position → answer string |
| `data/embeddings/clue_context_embeddings.npy` | (240,211, 1024) | 938 MB | Per clue row: word1_clue_context |
| `data/embeddings/clue_context_index.csv` | (240,211, 2) | ~5 MB | Maps row position → clue_id |

- **N_def** = 27,385 unique definition strings (after Step 2 cleanup)
- **N_ans** = 45,254 unique answer strings (after Step 2 cleanup)
- **N_rows** = 240,211 (rows from `clues_filtered.csv` after Step 2 cleanup — see `clue_context_phrases.csv`)
- **D** = 1024 (CALE-MBERT-en embedding dimension)
- **Total storage:** ~1.8 GB for all embedding files
- The three slots in definition/answer embeddings correspond to:
  `[0]` = allsense_avg (averaged across all WordNet synset contexts),
  `[1]` = common synset, `[2]` = obscure synset

**Mapping embeddings to clue rows:** For a given row in `clues_filtered.csv`,
look up its `definition_wn` string in `definition_index.csv` to get the row
position in `definition_embeddings.npy`, and similarly for `answer_wn`. The
clue-context embeddings are aligned directly with `clue_context_index.csv`
row order.

**Important: `keep_default_na=False` when loading index CSVs.** The word "nan"
(meaning grandmother) is a valid crossword definition and answer. Without this
flag, pandas interprets the string "nan" as `NaN`, silently corrupting the
index lookup. Always use `pd.read_csv(..., keep_default_na=False)` when
reading any CSV that contains the `word`, `definition_wn`, or `answer_wn`
columns.

### Step 3 Output

| File | Description |
|------|-------------|
| `data/features_all.parquet` | All 47 features + metadata for every clue row |

**Feature columns (47 total):**

**Context-Free Meaning (15):** Cosine similarities among embeddings that don't
involve clue context. Named as `cos_{emb_a}_{emb_b}`, e.g.:
- `cos_w1all_w2all`, `cos_w1all_w2common`, `cos_w1all_w2obscure`
- `cos_w1common_w2all`, `cos_w1common_w2common`, `cos_w1common_w2obscure`
- `cos_w1obscure_w2all`, `cos_w1obscure_w2common`, `cos_w1obscure_w2obscure`
- `cos_w1all_w1common`, `cos_w1all_w1obscure`, `cos_w1common_w1obscure`
- `cos_w2all_w2common`, `cos_w2all_w2obscure`, `cos_w2common_w2obscure`

**Context-Informed Meaning (6):** Cosine similarities involving
`word1_clue_context`:
- `cos_w1clue_w1all`, `cos_w1clue_w1common`, `cos_w1clue_w1obscure`
- `cos_w1clue_w2all`, `cos_w1clue_w2common`, `cos_w1clue_w2obscure`

**Relationship (22):**
- 20 boolean columns for two-hop WordNet relationship types (the originally
  planned 19 plus `hypernym_of_hyponym`; e.g., `wn_rel_synonym`,
  `wn_rel_hyponym`, `wn_rel_hypernym`, `wn_rel_hyponym_of_hypernym`, etc.)
- `wn_max_path_sim` (float, default 0.0)
- `wn_shared_synset_count` (int, default 0)

**Surface (4):**
- `surface_edit_distance` (int)
- `surface_length_ratio` (float)
- `surface_shared_first_letter` (int, 0/1)
- `surface_char_overlap_ratio` (float)

**Metadata columns (13, not features):**
- `clue_id`, `clue`, `surface`, `surface_normalized`, `definition`, `answer`,
  `definition_wn`, `answer_wn`, `def_answer_pair_id`, `answer_format`,
  `num_definitions`
- `def_num_usable_synsets`, `ans_num_usable_synsets` (for downstream
  stratification by polysemy level — see Decision 19)
